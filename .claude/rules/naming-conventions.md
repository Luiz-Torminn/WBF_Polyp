## Naming and Content-Language Rules

### Rule 73 — Name Code and Files in English

All code identifiers and all file and folder names must be in English. This covers variables, functions, hooks, components, types, interfaces, enums, constants, props, and module/file/folder names.

Do not name code or files in Portuguese. Prefer the English term over an anglicized or Portuguese spelling (e.g. `schedule` over `cronograma`, `user` over `usuario`). When a feature already uses a Portuguese name, follow the existing name for surgical changes, but flag it as cleanup rather than spreading the Portuguese naming further.
